The Files view provides a fast, keyboard-driven file browser with vim-style navigation and integrated command execution.

## Accessing the Files View

<CardGroup cols={2}>
  <Card title="Keyboard" icon="keyboard">

  </Card>
  <Card title="Tab Navigation" icon="arrow-right">
    Press `Tab` to cycle through views

## Navigation

### Vim-Style Movement

Filecast supports both arrow keys and vim-style keybindings for navigation.

<CodeGroup>
```text Vim Keys
j       Move down
k       Move up
l       Open file / Enter directory
h       Go to parent directory
```

```text Arrow Keys
↓       Move down
↑       Move up
→       Open file / Enter directory
←       Go to parent directory
```
</CodeGroup>

<Note>
Vim keys (`j`, `k`, `l`, `h`) work identically to arrow keys. Use whichever feels more comfortable.
</Note>

### Directory Navigation

From `src/ui/launcher.rs:261-301`, the Files view handles navigation:

<Tabs>
  <Tab title="Enter Directory">
    **Keys:** `→`, `l`, or `Enter`

    ```text
    Documents/              # Selected
    ├─ projects/
    ├─ notes.txt
    └─ README.md
    
    Press → or l or Enter → Navigate into Documents/
    ```

    When you enter a directory:
    - Selection resets to the first item
    - Directory contents load immediately
    - Current path updates in the header
  </Tab>

  <Tab title="Parent Directory">
    **Keys:** `←`, `h`, or `Backspace`

    ```text
    /home/user/Documents/projects/
    
    Press ← or h or Backspace → Navigate to /home/user/Documents/
    ```

    The parent directory is always available via the `..` entry at the top of the list.
  </Tab>

  <Tab title="Open File">
    **Keys:** `→`, `l`, or `Enter`

    ```text
    document.pdf            # Selected
    
    Press → or l or Enter → Opens in default application
    ```

    From `src/core/app.rs:316-325`, files are:
    - Logged to recent access history
    - Opened with the system's default application using `opener::open()`
  </Tab>
</Tabs>

### Command Execution

From `src/ui/launcher.rs:1104-1140`, commands:

1. **Execute synchronously** - UI waits for completion
2. **Run in current directory** - Uses `app.current_path`
3. **Save to history** - Stored in database with path context
4. **Display output** - Shows stdout/stderr
5. **Refresh directory** - Updates file list after execution

### Command History

When command input is **empty**, press `↑/↓` to browse history.

<Tabs>
  <Tab title="Browse">
    **Keys:** `↑` / `↓` (when input is empty)

    ```text
    Command History:
    > $ ls -la              /home/user/projects
    > $ git status          /home/user/projects
    > $ npm install         /home/user/projects/app
    ```

    Each entry shows:
    - Command text (left)
    - Directory where it was run (right, muted)
  </Tab>

  <Tab title="Execute from History">
    **Select command** → **Press Enter**

    The command re-executes in its **original directory**, not necessarily the current one.

    From `src/ui/launcher.rs:667-673`:

  </Tab>

  <Tab title="Click to Execute">

  </Tab>
</Tabs>

### Tab Completion

**Key:** `Tab` (while typing a command)

From `src/ui/launcher.rs:1142-1193`, tab completion:

- Completes **file and directory names**
- Works with **partial paths** (e.g., `src/co<Tab>`)
- Cycles through matches on repeated Tab presses
- Appends `/` to directories automatically

<CodeGroup>
```bash First Tab
$ cat Do<Tab>
→ $ cat Documents/
```

```bash Partial Path
$ cd proj/src/co<Tab>
→ $ cd proj/src/core/
```

```bash Multiple Matches
$ ls D<Tab><Tab><Tab>
Desktop/ → Documents/ → Downloads/ → Desktop/ ...
```
</CodeGroup>

### Exiting Command Mode

**Key:** `Escape`

From `src/ui/launcher.rs:237-241`:
- Closes command input
- Clears typed command
- Returns focus to file list

## Refresh Directory

**Key:** `r`

Reloads the current directory from disk. Useful after:
- Running commands that modify files
- External changes to the filesystem
- Creating/deleting files

From `src/ui/launcher.rs:303-305` and `src/core/app.rs:186-192`:

```rust
pub fn refresh_directory(&mut self) -> Result<()> {
    let entries = fs::read_directory(&self.current_path, self.show_hidden)?;
    self.file_list = entries;
    self.selected_index = 0;
    self.update_preview();
    Ok(())
}
```

```rust
            if path.is_dir() {

            } else {
                self.open_file(path.clone())?;
            }
        }

    }
}
```

## Tips

<AccordionGroup>
  <Accordion title="Use command mode for quick operations">
    Instead of switching to a terminal:
    - `c` → `git status` → `Enter`
    - `c` → `mkdir new-folder` → `Enter`
    - `c` → `touch file.txt` → `Enter`
    - Press `r` to refresh and see changes
  </Accordion>

  <Accordion title="Command history saves context">
    Commands remember the directory they were run in. This means you can:
    - Run `npm install` in a project directory
    - Navigate elsewhere
    - Re-run the same command later - it executes in the original directory
  </Accordion>

  <Accordion title="Tab completion is your friend">
    Long file names? Use Tab:
    - Start typing a few characters
    - Press Tab to complete
    - Keep pressing Tab to cycle through matches
  </Accordion>

  <Accordion title="Vim keys work everywhere">
    If you're a vim user, `hjkl` navigation works in Files view just like in vim:
    - `h` = left (parent)
    - `j` = down
    - `k` = up  
    - `l` = right (open/enter)
  </Accordion>
</AccordionGroup>
